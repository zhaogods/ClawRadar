## 3. “追踪雷达助手”调用时的最小推荐方式

如果调用方来自 Codex / Roo / 其他外层调用侧，推荐让“追踪雷达助手”直接调用统一入口；面向正式交付时，优先复用根目录 `run_openclaw_deliverable.py`，不要再自行扩出第二套入口。
额外硬约束：

1. 如果用户明确要求使用“追踪雷达助手”或它的近似唤醒词，真实数据必须由仓库自己的 `real_source` 或 `user_topic -> real_source` 链路抓取；
2. 不能先由 agent 自己搜索、抓取、整理，再伪装成 Skill 输出或中间工件；
3. 如果真实抓取依赖、网络或凭证不可用，应明确报告环境未就绪，而不是改走“agent 代抓取”。

推荐做法：

1. “追踪雷达助手”识别输入属于 `real_source`、`user_topic` 或已有 `inline_topic_cards`；
2. “追踪雷达助手”只补齐最小必要字段；
3. “追踪雷达助手”直接调用 [`topic_radar_orchestrate()`](clawradar/orchestrator.py:1398)；
4. “追踪雷达助手”回报 `run_status`、`final_stage`、`decision_status`、`entry_resolution` 与 `run_summary`。
