# 深度爬取能力增强实施报告

生成时间：2026-04-30

## 1. 报告目的

基于《四引擎能力边界与主线调用分析报告》的结论——当前 `clawradar/` 编排器仅调用 `radar_engines/` 各引擎约 15% 的浅层接口——本报告记录从"浅层热点聚合"升级为"热点发现→关键词提取→全平台社媒深爬→多维度搜索增强→证据包注入"完整深度爬取链路的实施过程。

---

## 2. 改造前状态

### 2.1 旧爬取链路

```
input → MindSpider(get_popular_news 12源热点)
      → QueryEngine/MediaEngine(search_last_week 单次搜索)
      → topic_candidates → score → write → deliver
```

### 2.2 引擎利用度

| 引擎 | 已使用 | 未使用 |
|------|--------|--------|
| **MindSpider** | `BroadTopicExtraction.get_popular_news()` | `DeepSentimentCrawling`（7平台社媒深爬）、`DatabaseManager`（持久化）、`TopicExtractor`（AI关键词提取） |
| **QueryEngine** | `search_news_last_week()` | `search_news_last_24_hours()`、`search_images_for_news()`、`DeepSearchAgent`（4阶段深度搜索） |
| **MediaEngine** | `search_last_week()` | `search_for_structured_data()`、Anspire 搜索提供商 |
| **ReportEngine** | `generate_report()` | 完整（已充分使用） |

### 2.3 关键缺失

- `DeepSentimentCrawling` 下的 `MediaCrawler/` 子模块未初始化（git submodule 不存在）
- `real_source.py` 中 `db_manager=None` 硬编码，热点数据不持久化
- `keyword_manager.py:327` 调用不存在的 `distribute_keywords_by_platform()` 方法
- 搜索结果来自单时间窗口（一周），无图片、无结构化数据
- 搜索引擎关键词仅用用户输入，未利用 BroadTopic AI 提取的精准关键词

---

## 3. 改造后链路

```
input → BroadTopic(12源热点+AI关键词提取) [持久化入库]
      → [按评分阈值触发] DeepSentiment(7平台社媒深爬)
      → QueryEngine(Tavily 多时间窗口+图片搜索)
      → MediaEngine(Bocha/Anspire 多窗口+结构化数据)
      → [跨引擎关键词补搜]
      → topic_candidates(附社媒证据+图片+结构化数据) → score → write → deliver
```

8 阶段管线：`entry → crawl → ingest → topics → score → deep_crawl(新) → write → deliver → notify`

## 4. 分阶段实施

### 4.1 Phase 1：MediaCrawler 子模块初始化

**问题**：`radar_engines/MindSpider/DeepSentimentCrawling/MediaCrawler/` 目录不存在，`PlatformCrawler.__init__()` 启动即抛 `FileNotFoundError`。

**操作**：
- 从 `ghfast.top` 镜像克隆 `NanmiCoder/MediaCrawler`
- 移除嵌套 `.git` 作为常规文件集成（因网络限制无法使用 git submodule）

**涉及文件**：
- `radar_engines/MindSpider/DeepSentimentCrawling/MediaCrawler/`（新增目录，含 `main.py`、`config/`、`media_platform/`、`database/` 等）

**验证**：`PlatformCrawler.__init__()` 通过，7 平台检测正常。

### 4.2 Phase 2：MindSpider 深度爬取接入编排器

#### 4.2.1 修复 keyword_manager.py Bug

`keyword_manager.py:327` 的 `__main__` 测试块调用了不存在的 `distribute_keywords_by_platform()` 方法。

**修复**：替换为实际存在的 `get_keywords_for_platform(platform, max_keywords=10)`。

#### 4.2.2 启用数据库持久化

`_collect_mindspider_news()` 新增 `persist` 参数：
- 当 `persist=True` 时，正常实例化 `DatabaseManager` 并赋值给 `collector.db_manager`
- 调用 `collector.collect_and_save_news()` 持久化热点数据
- 调用 `TopicExtractor.extract_keywords_and_summary()` 提取 AI 关键词
- 返回类型从 3 元组扩展为 4 元组：`(results, source_names, base_url, extracted_keywords)`

```python
def _collect_mindspider_news(
    source_ids: Sequence[str],
    *,
    persist: bool = False,
) -> Tuple[List[Dict[str, Any]], Dict[str, str], str, List[str]]:
```

#### 4.2.3 新增深爬桥接函数

```python
def _run_deep_sentiment_crawling(
    platforms: List[str] | None = None,
    max_keywords: int = 50,
    max_notes: int = 50,
    test_mode: bool = False,
) -> Dict[str, Any]:
```

加载 `DeepSentimentCrawling` 模块，实例化并调用 `run_daily_crawling()`，返回各平台爬取统计。

#### 4.2.4 编排器新增 DEEP_CRAWL 阶段

在 Score 之后、Write 之前插入 `DEEP_CRAWL` 阶段。

**触发条件**：
- `entry_options.deep_crawl.enabled = true`（默认关闭，向后兼容）
- 存在 `decision == "publish_ready"` 的选题

**配置项**：

| 字段 | 默认值 | 说明 |
|------|--------|------|
| `enabled` | `false` | 显式开启 |
| `platforms` | `["xhs","dy","ks","bili","wb","tieba","zhihu"]` | 目标平台 |
| `max_keywords` | `50` | 每平台关键词数 |
| `max_notes` | `50` | 每平台采集笔记数 |
| `test_mode` | `false` | 测试模式 |

**涉及文件**：
- `clawradar/real_source.py:140-264`（`_run_deep_sentiment_crawling`、`_collect_mindspider_news` 增强）
- `clawradar/orchestrator.py:2281-2310`（DEEP_CRAWL 阶段）
- `radar_engines/MindSpider/DeepSentimentCrawling/keyword_manager.py:327`（Bug 修复）

### 4.3 Phase 3：搜索引擎多维度增强

#### 4.3.1 多时间窗口搜索

三个提供商（Tavily/Bocha/Anspire）均启用双时间窗口并发搜索：

```python
_TIME_WINDOW_WEIGHTS = {"24h": 1.0, "week": 0.7, "month": 0.4}

windows = [
    ("24h", lambda: client.search_news_last_24_hours(query)),
    ("week", lambda: client.search_news_last_week(query)),
]
```

搜索结果按时间衰减权重合并，统一通过 `_normalize_search_item()` 标准化字段。

#### 4.3.2 图片搜索

`_search_with_tavily()` 在文本搜索后追加 `search_images_for_news(query)`，图片结果附加置信度 `≥0.55`。

#### 4.3.3 结构化数据搜索

`_search_with_bocha()` 追加 `search_for_structured_data(query)`，结构化数据附加置信度 `≥0.68`。

#### 4.3.4 搜索结果标准化

新增 `_normalize_search_item()`：统一 Tavily/Bocha/Anspire 三提供商输出格式，增加 `time_window`、`time_weight` 字段。

新增 `_map_topic_search_item_to_candidate()` 时间权重调整逻辑：

```python
time_weight = float(item.get("time_weight") or 1.0)
adjusted_confidence = min(0.85, base_confidence * time_weight + 0.10)
if item.get("is_structured"):
    adjusted_confidence = max(adjusted_confidence, 0.68)
if item.get("is_image"):
    adjusted_confidence = max(adjusted_confidence, 0.55)
```

**涉及文件**：
- `clawradar/real_source.py`（`_search_with_tavily`、`_search_with_bocha`、`_search_with_anspire`、`_normalize_search_item`、`_map_topic_search_item_to_candidate`）
- `clawradar/scoring.py`（消费新字段）

### 4.4 Phase 4：跨引擎关键词协同

`_search_topic_news()` 新增 `extra_queries` 参数，接收 BroadTopic AI 提取的关键词作为补充搜索词（上限 5 个）。

```

↓
对每个 extra_query 执行轻量搜索（limit//3）
↓
标记 source_type:enriched，time_weight × 0.8
↓
去重后追加到 merged_items
```

返回类型从 4 元组扩展为 5 元组：
```python
def _search_topic_news(
    context, *, limit, extra_queries: Sequence[str] = (),
) -> Tuple[List, str, List, List, List]:
    # merged_items, query, failed_sources, applied_source_ids, enrichment_items
```

## 7. 使用方式

### 7.1 深爬开关（API 调用）

```json
{
  "entry_options": {
    "input": { "mode": "real_source", "source_ids": ["weibo"], "limit": 5, "persist": true },
    "deep_crawl": {
      "enabled": true,
      "platforms": ["wb", "zhihu", "xhs"],
      "max_keywords": 30,
      "max_notes": 30
    }
  }
}
```

### 7.2 MindSpider 独立测试

```bash
cd radar_engines/MindSpider
python main.py --broad-topic                      # 热点提取 → 产生关键词入库
python main.py --deep-sentiment --test            # 深爬测试模式
python main.py --complete --test                  # 完整链路测试
```

### 7.3 ClawRadar 全链路

```bash
python run_clawradar_deliverable.py \
  --input-mode real_source --source-ids weibo --limit 5 \
  --entry-options '{"deep_crawl":{"enabled":true,"platforms":["wb","zhihu"]}}'
```
